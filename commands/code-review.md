# 代码审查 (Code Review)

对未提交的更改进行全面的安全和质量审查：

1. 获取更改的文件：`git diff --name-only HEAD`

2. 对于每个更改的文件，检查：

**安全问题 (严重 - CRITICAL):**
- 硬编码凭据、API 密钥、令牌
- SQL 注入漏洞
- XSS 漏洞
- 缺失输入验证
- 不安全的依赖
- 路径遍历风险

**代码质量 (高 - HIGH):**
- 函数 > 50 行
- 文件 > 800 行
- 嵌套深度 > 4 层
- 缺失错误处理
- console.log 语句
- TODO/FIXME 注释
- 公共 API 缺失 JSDoc

**最佳实践 (中 - MEDIUM):**
- 变异模式 (应使用不可变数据)
- 代码/注释中的 Emoji 使用
- 新代码缺失测试
- 可访问性问题 (a11y)

3. 生成报告包含：
   - 严重性：严重 (CRITICAL), 高 (HIGH), 中 (MEDIUM), 低 (LOW)
   - 文件位置和行号
   - 问题描述
   - 建议修复

4. 如果发现严重或高优先级问题，阻止提交

永远不要批准带有安全漏洞的代码！
