# /learn - 提取可复用模式

分析当前会话并提取任何值得保存为技能的模式。

## 触发条件

当您解决了一个非平凡的问题后，在会话的任何时候运行 `/learn`。

## 提取什么

寻找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 怎么修复的？
   - 这对于类似错误是否可复用？

2. **调试技术**
   - 不明显的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方法 (Workarounds)**
   - 库的怪癖
   - API 限制
   - 特定版本的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建技能文件：

```markdown
# [描述性模式名称]

**Extracted:** [日期]
**Context:** [简要描述何时适用]

## 问题
[这解决了什么问题 - 具体点]

## 解决方案
[模式/技术/变通方法]

## 示例
[代码示例（如果适用）]

## 何时使用
[触发条件 - 什么应该激活此技能]
```

## 流程

1. 审查会话以寻找可提取的模式
2. 识别最有价值/可复用的见解
3. 起草技能文件
4. 请求用户确认保存
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不要提取琐碎的修复（拼写错误、简单的语法错误）
- 不要提取一次性问题（特定的 API 中断等）
- 专注于能为未来会话节省时间的模式
- 保持技能专注 - 每个技能一个模式
